Preact in pictures

A talk at the ReactJS Denver meetup, walking through what Preact does internally to render and diff a simple application — and what that mental model tells us about React.

The talk in two parts

I gave a talk at tonight’s ReactJS Denver meetup about Preact. The thirty-minute video is embedded below; the slides are higher-resolution if you’d rather flip through them at your own pace1.

  • Part 1 describes Preact, why you might want to use it, and the tradeoffs you have to make by not using full React.
  • Part 2 walks through what Preact is doing internally to render and diff a simple application.
To many people, the virtual DOM and component model seem like deep mysteries that mere mortals shouldn’t try to understand. They aren’t.

It’s a reasonably simple algorithm with a few fiddly little details. While React is implemented somewhat differently, knowing Preact’s internals is still a good first mental model for approaching it.

The core diff loop

The heart of the renderer is roughly this — a recursive walk that reconciles a virtual node against the DOM node it last produced, returning the (possibly new) DOM node to its parent2:

// Preact: diff a vnode against an existing dom node.
function diff(dom, vnode, parent) {
  if (typeof vnode === 'string') {
    return diffText(dom, vnode);
  }
  if (typeof vnode.nodeName === 'function') {
    return buildComponentFromVNode(dom, vnode);
  }
  return innerDiffNode(dom, vnode);
}

Everything else in the codebase is in service of those three branches: handling text nodes, building up component instances, and walking the children of an element.
