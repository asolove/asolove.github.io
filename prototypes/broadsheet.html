Preact, in pictures.

A thirty-minute walk through what a tiny React-alike does inside — and what its small, legible algorithm can teach us about the larger one.

By Adam Solove Reading time · 4 minutes Filed under JS · UI

I gave a talk at tonight’s ReactJS Denver meetup about Preact. The video is embedded below the body; the slides are higher resolution if you’d rather flip through them at your own pace.1

The talk comes in two parts.

  • Part 1 describes Preact, why you might want to use it, and the tradeoffs you make by not using full React.
  • Part 2 walks through what Preact is doing internally to render and diff a simple application.
The virtual DOM is not a deep mystery. It is a small algorithm with a few fiddly details.

To many people, the virtual DOM and component model seem like deep mysteries that mere mortals shouldn’t try to understand. As I hopefully demonstrated, this isn’t true. It’s a reasonably simple algorithm with a few fiddly little details. While React is implemented somewhat differently, knowing Preact’s internals is still a good first mental model for approaching it.

The core diff loop

The heart of the renderer is roughly this: a recursive walk that reconciles a virtual node against the DOM node it last produced, returning the (possibly new) DOM node to its parent.2

// Preact: diff a vnode against an existing dom node.
function diff(dom, vnode, parent) {
  if (typeof vnode === 'string') {
    return diffText(dom, vnode);
  }
  if (typeof vnode.nodeName === 'function') {
    return buildComponentFromVNode(dom, vnode);
  }
  return innerDiffNode(dom, vnode);
}

Everything else in the codebase is in service of those three branches: handling text nodes, building up component instances, and walking the children of an element. Once you can hold those branches in your head, the rest is bookkeeping.
